# 03: PER-ORGAN README TEMPLATES

Seven ready-to-use templates, one for each organ type. Customize per repo, don't skip sections.

---

## ORGAN-I (Theory) — Template

```markdown
---
title: [Repo Name]
organ: ORGAN-I (Theory)
status: ACTIVE
last_updated: YYYY-MM-DD
author: Your Name
---

# [Repo Name]

## Problem Statement
[1-2 paragraphs: What intellectual gap does this address? Why does this framework matter? Who benefits?]

## Core Concepts
[3-5 core ideas, each 2-3 paragraphs. This is the heart of theory documentation.]

### Concept 1: [Name]
[Explanation + context + examples]

### Concept 2: [Name]
[Explanation + context + examples]

[Continue for 3-5 concepts...]

## Related Work
[How does this compare to academic/artistic precedent? List 3-5 similar projects + how yours differs.]

## Installation & Usage
[If code: Clear setup instructions. Include dependencies + versions. Provide runnable example.]

## Examples
[2-3 concrete examples showing the theory in practice.]

### Example 1: [Name]
[Code + explanation]

### Example 2: [Name]
[Code + explanation]

## Downstream Implementation
[Which ORGAN-II repos implement this theory? Links + brief descriptions.]

## Validation
[How do we know this framework works? What tests or examples prove it?]

## Roadmap
[Future directions for this framework. What could extend this?]

## Cross-References
[Links to other ORGAN-I repos this relates to. Upstream dependencies?]

## Contributing
[How others can contribute to this theory work.]

## License
[License info]

## Author & Contact
[Your name, email, social media (if applicable)]
```

**Expected length:** 3,000+ words | **Generation:** ~72K TE | **Review:** ~15-20 min per AI-generated draft

[High-level system design. Key components. Scalability notes.]

[How was this built? What problems did you solve? Lessons learned?]
